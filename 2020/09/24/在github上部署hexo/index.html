在github上部署hexo

安装node.js

node.js官网

安装完毕后，在cmd命令窗口中输入node -v 和 npm -v，显示正常的版本号说明安装成功

可以安装cnpm(淘宝镜像)，加快下载速度，命令
npm i -g cnpm --registry=https://registry.npm.taobao.org

安装hexo

npm install hexo-cli -g

安装完毕执行hexo -v确认安装成功，该命令会显示hexo的版本和所依赖的包

生成本地博客文件

  1. 创建一个空白文件夹，作为博客根目录，在该根目录下，初始化博客，命令hexo init(初始化需要时间，请耐心等待)

  2. 启动项目，命令hexo serve

    直到看到 Hexo is running at http://localhost:4000 .Press Ctrl+C to stop. 证明启动成功

    在浏览器地址栏输入http://localhost:4000即可访问博客

常用的hexo命令

  • 新建博客，hexo new '博客名'

    生成的博客会放置在source文件夹中

  • 生成静态文件，hexo generate (会在根目录生成public文件夹)

  • 部署到git上，hexo deploy (需要配置)

    安装依赖包npm install hexo-deployer-git --save
    在根目录的_config.yml中的deploy中配置如下信息

安装git

git官网

安装完毕后，鼠标右键菜单会多出来两个跟git有关的选项，选择Git Bash Here

生成和设置公钥

  • 配置git用户信息

    git config --global user.name "GitHub用户名"

    git config --global user.email "GitHub邮箱"

  • 生成命令ssh-keygen -t rsa -C "Github邮箱"

  • 生成之后，到C:\Users\Administrator\.ssh中找到id_rsa.pub文件,id_rsa时私钥，不要泄露给别人

  • 进入github官网，点击自己头像下的Settings进入设置页面

  • 点击SSH and GPG keys进行配置公钥(SSH key)，把id_rsa.pub中的内容复制进去即可

  • 新建github仓库，名字为你的github名.github.io，名字格式必须正确

  • 会生成一个链接https://你的github名.github.io/，在浏览器中输入链接即可访问

安装或下载NexT主题插件

在根目录执行git clone https://github.com/theme-next/hexo-theme-next themes/next即可下载最新主题插件，

文件会放置在themes/next中

修改主题时，打开根目录的_config.yml文件，找到theme设置为next即可

在next文件夹中的_config.yml的``scheme`位置，可以选择主题

生成静态文件

直接使用命令hexo generate

部署至github

配置好对应的配置后，执行hexo deploy即可，等待几分钟后，刷新链接https://你的github名.github.io/

写博客

  • 文件夹scaffolds存放模板文件
    1. draft.md
    2. page.md，博客页面文件
    3. post.md
